# ADR-001: Usar PostgreSQL como Banco de Dados

## Status

Aceita

## Contexto

Necessário consistência de dados (ACID) e segurança de gravação, NoSQL não oferece garantias suficientes.

## Decisão

Adotaremos PostgreSQL como banco de dados principal da aplicação.

## Consequências

### Positivas
- Garantias ACID
- Dados previsíveis e estruturados
- Consultas e relatórios complexos

### Negativas
- Escalabilidade horizontal limitada - SQL tradicional é mais difícil de escalar em múltiplos nós
- Menos flexível para mudanças de schema - alterações na estrutura de dados requerem migrações complexas
