# RFC-001 — Escolha da AWS como Provedor de Nuvem

**Status:** Aceito  
**Data:** 2026-05-24  
**Autor:** Time de Engenharia  

---

## Contexto

A aplicação é um backend Node.js/TypeScript containerizado, com banco de dados PostgreSQL, orquestração via Kubernetes e pipeline de CI/CD no GitHub Actions. É necessário escolher um provedor de nuvem para hospedar a infraestrutura de produção.

## Decisão

Adotar a **AWS (Amazon Web Services)** como provedor principal de nuvem.

## Justificativa

### Aderência técnica

| Necessidade | Serviço AWS |
|---|---|
| Kubernetes gerenciado | EKS (Elastic Kubernetes Service) |
| Banco de dados PostgreSQL | RDS for PostgreSQL |
| Registro de imagens Docker | ECR (Elastic Container Registry) |
| CI/CD (GitHub Actions) | IAM OIDC + roles para deploy sem credenciais estáticas |
| Auto Scaling (HPA configurado) | Cluster Autoscaler para EKS |
| Secrets e variáveis de ambiente | AWS Secrets Manager / Parameter Store |

Os manifestos Kubernetes existentes em `/k8s/` são compatíveis com EKS sem alterações estruturais.

### Maturidade e ecossistema

A AWS é o provedor com maior market share global e o mais maduro em serviços gerenciados de Kubernetes. A documentação, suporte da comunidade e integrações com ferramentas open-source (Helm, Terraform, ExternalDNS, etc.) são amplos e bem estabelecidos.

### Modelo de custo previsível

O EKS oferece preço fixo por cluster com cobrança por nó EC2, permitindo dimensionamento controlado via HPA e Cluster Autoscaler já previstos na arquitetura. Instâncias Spot podem ser usadas para reduzir custo em ambientes não-críticos.

Além de termos o AWS Academy por parte da FIAP disponibilizando $50 de créditos para utilizarmos no projeto. Uma plataforma de custos previsíveis como a AWS no ajudaria a planejar melhor os custos da infraestrutura provisionada e da aplicação rodando em si.

### Segurança e conformidade

A AWS possui certificações SOC 2, ISO 27001 e PCI DSS, além de suporte nativo a IAM com políticas granulares, criptografia em repouso e em trânsito, e VPC para isolamento de rede.

### Adoção no mercado brasileiro

Alta disponibilidade de profissionais com certificação AWS no mercado, reduzindo fricção em contratações e transferência de conhecimento.

## Alternativas Consideradas

| Provedor | Motivo da não-escolha |
|---|---|
| GCP | Kubernetes mais maduro (GKE), mas menor adoção no Brasil e ecossistema menor para integrações empresariais locais |
| Azure | Boa integração com Microsoft stack, mas overhead de configuração maior para workloads nativamente Linux/open-source |

## Consequências

- A infraestrutura será provisionada via IaC (Terraform ou CDK) na conta AWS dedicada ao projeto.
- Decisões futuras de serviços (filas, cache, storage) devem priorizar serviços gerenciados AWS antes de introduzir dependências externas.
